### 3. Composite Constraints (Составные ограничения уникальности)

**Статус:** ✅ **ПОДДЕРЖИВАЕТСЯ**

**Синтаксис:**
```cypher
CREATE CONSTRAINT composite_test_id_domain_unique
FOR (n:CompositeTest)
REQUIRE (n.id, n.domain) IS UNIQUE
```

**Результаты тестирования:**
- ✅ Создание constraint успешно
- ✅ MERGE идемпотентен: второй MERGE обновляет существующий узел
- ✅ CREATE блокирует дубликаты: constraint violation при попытке создать узел с теми же (id, domain)

**Пример идемпотентности:**
```cypher
-- Первый MERGE создает узел
MERGE (n:CompositeTest {id: 100, domain: 'test.com'})
ON CREATE SET n.name = 'First'

-- Второй MERGE обновляет существующий узел (не создает дубликат)
MERGE (n:CompositeTest {id: 100, domain: 'test.com'})
ON MATCH SET n.name = 'Second'
```

**Вывод:**
- Composite constraints полностью поддерживаются в Community Edition
- Идемпотентность MERGE работает корректно с composite constraints
- Можно использовать для обеспечения уникальности комбинаций свойств
